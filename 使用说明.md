# 使用说明

## 📱 访问应用

部署完成后，打开浏览器访问你的 Cloudflare Pages 网址。

## ⚠️ 重要提示

**所有数据仅存储在浏览器本地**，不会上传到任何服务器。请务必：
- ⚡ 定期使用「导出数据」功能备份交易记录
- ⚡ 将备份文件保存在安全位置（云盘、GitHub等）
- ⚡ 清除浏览器数据会导致记录永久丢失

## ✍️ 记录交易

### 添加买入记录
1. 选择"买入"
2. 填写交易日期
3. 输入买入价格（美元）
4. 输入买入数量（BTC）
5. 可以添加当时的投资思考
6. 点击"添加记录"

### 添加卖出记录
操作同上，选择"卖出"即可

## 📊 查看收益

仪表板实时显示：
- **实时价格**：自动每分钟更新
- **持仓数量**：当前持有的比特币总量
- **平均成本**：你的平均买入价格
- **总投入**：净投入金额（买入支出 - 卖出收入）
- **当前市值**：按实时价格计算的持仓价值
- **账面收益**：显示盈亏金额和收益率

### 收益计算公式

```
持仓数量 = 所有买入数量 - 所有卖出数量
平均成本 = 总买入金额 ÷ 总买入数量
总投入 = 总买入金额 - 总卖出金额
当前市值 = 持仓数量 × 当前BTC价格
账面收益 = 当前市值 - 总投入
收益率 = (账面收益 ÷ 总投入) × 100%
```

## 💾 数据管理

### 导出数据
点击「导出数据」按钮，会下载一个 JSON 文件，包含所有交易记录。

**⚠️ 强烈建议：**
- 每次添加重要交易后立即导出备份
- 将备份文件保存在多个位置（电脑、云盘、GitHub等）
- 定期检查备份文件是否完整

### 导入数据
1. 点击「导入数据」
2. 选择之前导出的 JSON 文件
3. 确认导入（会覆盖现有数据）

### 测试数据
项目包含 `example-data.json` 示例文件，可以导入体验功能。

## 📱 多设备使用

### 方法1：导出导入
1. 设备A导出数据
2. 通过云盘或邮件传输
3. 设备B导入数据

### 方法2：GitHub同步
1. 创建私有 GitHub 仓库
2. 定期上传导出的 JSON 文件
3. 其他设备从 GitHub 下载并导入

## ⚠️ 注意事项

### 数据安全
- ✅ 数据只存储在浏览器本地
- ✅ 不会上传到任何服务器
- ⚠️ **清除浏览器数据会永久丢失所有记录**
- ⚠️ 换电脑/浏览器必须通过「导入数据」恢复
- ⚠️ 浏览器隐私模式下数据不会持久保存
- ⚠️ 重装系统或浏览器前务必先导出数据

### 隐私保护
- 不在公共电脑使用
- 定期清理下载的备份文件
- 考虑对敏感数据加密

### 投资风险
- 本应用仅用于记录
- 不构成投资建议
- 投资有风险，决策需谨慎

## 🆘 常见问题

### Q: 价格不更新怎么办？
A: 检查网络连接。国内用户可能需要代理访问 Coinbase API。

### Q: 数据丢失了怎么办？
A: 如果有备份的 JSON 文件，可以通过"导入数据"恢复。

### Q: 能记录其他加密货币吗？
A: 当前版本只支持比特币。如需其他币种，需修改代码。

### Q: 手机能用吗？
A: 可以！界面是响应式设计，支持手机和平板。

### Q: 数据能同步到云端吗？
A: 当前版本不支持自动云端同步。需要通过「导出数据」手动备份到云盘。
